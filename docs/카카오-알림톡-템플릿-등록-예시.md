# 카카오 알림톡 템플릿 등록 예시 (강조 표기형)

내용·버튼 컨셉(무엇을 제공할지, 버튼 정리)은 [카카오-알림톡-내용-버튼-컨셉.md](./카카오-알림톡-내용-버튼-컨셉.md)를 참고하세요.

- **템플릿 유형:** 강조 표기형으로 통일
- **템플릿 코드(ID)**는 아래와 **완전 동일**하게 입력 (API에서 사용, 영문 유지)
- **템플릿 이름**은 채널 관리자에서 찾기 쉽도록 한글로 입력 (아래 표 참고)
- 변수 형식: `#{변수명}` (이름·개수 변경 금지)
- **내용에 변수 추가** 모달에서 넣을 때는 `#{}` 없이 **변수명만** 입력 (예: `고객명`, `링크`)
- **강조 표기 보조문구에는 변수 사용 불가** → 보조문구는 고정 문구만 입력 (아래 표 참고)

| 부가서비스 주문 접수 완료 (고객) | 고객명, 서비스명, 금액, 링크 |

## 공통

- 알림톡은 정보 전달용. 마케팅·할인·이벤트 유도 문구 반려 가능.
- 버튼 최소 1개. 웹링크 PC/모바일 동일 URL.
- 메시지 15자 이상 ~ 1000자 이하.
- **수신거부 안내**: 모든 고객용 템플릿 본문 하단에 아래 문구를 넣을 것:  
  `알림 수신을 원하지 않으시면 카카오톡 채널을 차단해 주세요.`
- **사용자의 의견 (검수자 참고)**: 카카오 등록 시 "카카오측 검수자가 참고해야 할 사용자의 의견"란에 아래 각 템플릿별 문구를 **그대로 붙여넣기**하세요. 수신자에게 보이지 않으며, 부가정보는 변수 사용 불가이므로 변수 설명은 이 의견란에 작성합니다.

## 버튼 종류 및 설정 방법 (새 버튼 추가)

카카오 채널 관리자에서 **알림톡 버튼** 추가 시 아래 선택지가 있습니다. 우리 템플릿은 모두 **웹링크**만 사용합니다.

### 버튼 종류 선택 (선택지)

| 선택지 | 용도 | 우리 템플릿 사용 |
|--------|------|------------------|
| 채널추가 | 카카오톡 채널 친구 추가 | 사용 안 함 |
| **웹링크** | 버튼 클릭 시 웹페이지 열기 | **전부 이거 사용** |
| 앱링크(URL Scheme) | 앱 실행/딥링크 | 사용 안 함 |
| 배송조회 | 배송 조회 링크 | 사용 안 함 |
| 봇키워드 | 챗봇 키워드 호출 | 사용 안 함 |
| 메시지전달 | 메시지 전달 | 사용 안 함 |
| 상담전화 | 전화 걸기 | 사용 안 함 |
| 봇전환 | 봇 플로우 전환 | 사용 안 함 |

### 웹링크 버튼 입력 필드

**버튼 종류**에서 **웹링크** 선택 후:

| 필드 | 필수 | 입력 방법 |
|------|------|------------|
| **버튼명** | ✓ | 사용자에게 보이는 글자 (예: 대시보드 바로가기, 보고서 확인하기) |
| **Mobile 링크** | ✓ 필수 | 실행할 URL. 변수 쓸 때는 **프로토콜 고정**으로 입력. |
| **PC 링크** | 선택 | 동일 URL이면 Mobile과 같게 넣거나 비워도 됨. |
| **외부 브라우저로 링크 열기** | 선택 | 체크 시 카카오톡 앱이 아닌 사용자 브라우저에서 열림. (보통 체크 해제) |

**URL에 변수 넣는 규칙 (프로토콜 필수)**

- 안내: "최대 500자 이내, https:// 또는 http:// 프로토콜 필수, 변수사용가능" → **변수 쓸 때도 템플릿에 `https://` 를 반드시 넣는 게 맞음.**
- **권장 입력**: 모바일/PC 링크에 **`https://#{링크}`** (또는 `https://#{자세히보기}`, `https://#{승인하기}`) 로 입력.
- 그러면 API에서 보내는 `링크` 값은 **프로토콜 제외** (예: `www.onemarketing.kr/dashboard`) 로 보내면 됨. 우리 앱은 이렇게 보내도록 되어 있음.
- 예시: `https://#{링크}`, `https://#{자세히보기}`, `https://nurigo.net` (고정 URL).
- 최대 500자, `http://` 또는 `https://` 필수, 변수 사용 가능.

**아래 각 템플릿의 "버튼" 행**은 모두 **웹링크**로 등록하고, 표에 적힌 **버튼명**과 **Mobile/PC URL**만 그대로 넣으면 됩니다.

## 발송 트리거 (발송 시점) — 앱에서 이미 세팅됨

카카오/솔라피에는 “언제 보낼지” 설정이 없습니다. **우리 앱(코드)**에서 아래 시점에 API를 호출해 발송합니다.

| 템플릿 코드 | 발송 시점 | 비고 |
|-------------|-----------|------|
| `TPL_welcome` | 관리자가 **가입 승인** 시 | `admin/signups` 승인 시 |
| `TPL_report_published` | **보고서 발행** 시 또는 관리자 수동 발송 | 보고서 에디터에서 발행 / 알림 수동 발송 |
| `TPL_action_status` | **실행 항목 상태 변경** 시 또는 관리자 수동 발송 | execution 상태 변경 / 알림 수동 발송 |
| `TPL_event_reminder` | **일정 리마인더** 시 또는 관리자 수동 발송 | (추후 자동 스케줄) / 알림 수동 발송 |
| `TPL_password_reset` | (추후) **비밀번호 재설정** 요청 시 | 아직 호출 코드 없음 |
| `TPL_mon_review` | **매주 월요일 09:30 (KST)** 자동 | Vercel Cron |
| `TPL_wed_budget` | **매주 수요일 14:00 (KST)** 자동 | Vercel Cron |
| `TPL_thu_proposal` | **매주 목요일 16:00 (KST)** 자동 | Vercel Cron |
| `TPL_addon_order_admin` | 고객이 **부가서비스 주문** 시 | 관리자에게 발송 |
| `TPL_addon_order_client` | 고객이 **부가서비스 주문** 시 | 고객에게 주문 접수 확인 |

→ **트리거는 코드에 이미 연결되어 있음.** 템플릿만 카카오에 등록·승인하면 해당 시점에 자동 발송됩니다.

## 매직링크 현황 (고객용 링크 = 모두 매직링크)

**원칙**: 알림톡에 넣는 모든 고객용 링크는 **매직링크**(토큰 URL). 클릭 시 로그인 없이 또는 토큰 검증 후 해당 화면으로 이동.

| 템플릿 | 버튼/링크 | 매직링크 여부 | 비고 |
|--------|------------|----------------|------|
| TPL_welcome | 대시보드 바로가기 | ✅ 매직링크 | `/portal/v/{token}` → 오버뷰 이동 |
| TPL_report_published | 보고서 확인하기 | ✅ 매직링크 | `/report/v/{token}` (보고서 상세) |
| TPL_action_status | 실행 현황 보기 | ✅ 매직링크 | `/portal/v/{token}` → 실행 탭 이동 |
| TPL_event_reminder | 일정 보기 | ✅ 매직링크 | `/portal/v/{token}` → 타임라인 이동 |
| TPL_password_reset | 비밀번호 설정 | ✅ 매직링크 | 재설정 토큰 URL (추후 구현) |
| TPL_mon_review | 자세히 보기 | ✅ 매직링크 | `/report/v/{token}` (주간 알림 상세) |
| TPL_wed_budget | 자세히 보기 | ✅ 매직링크 | `/report/v/{token}` (주간 알림 상세) |
| TPL_thu_proposal | 자세히 보기 / 승인하기 | ✅ 매직링크 | `/report/v/{token}`, `/report/approve/{token}` |
| TPL_addon_order_admin | 주문 확인 | 고정 URL | 관리자 전용, 로그인 필요 |
| TPL_addon_order_client | 주문 내역 보기 | ✅ 매직링크 | `/portal/v/{token}` → 오버뷰 이동 |

## 4. TPL_event_reminder

## 등록 시 체크

1. **유형** → 강조 표기형 선택
2. **템플릿 코드** → 위 표의 TPL_xxx 그대로 입력 (API에서 사용)
3. **템플릿 이름** → 위 표의 한글 이름 입력 (관리자 화면에서 찾기 쉽게)
4. **강조 표기 제목 / 보조문구 / 내용** → 표에서 복사. 보조문구에는 변수 넣지 말 것 (변수 사용 불가). 내용의 변수명 `#{…}` 변경 금지.
5. **사용자의 의견 (검수자 참고)** → 각 템플릿의 "검수자 참고 의견" 문구를 **그대로** 붙여넣기 (수신자에게 보이지 않음, 부가정보는 변수 사용 불가이므로 변수 설명은 여기 작성)
6. **버튼** → **+ 버튼 추가** 클릭 후:
   - **버튼 종류 선택** → **웹링크** 선택
   - **버튼명** → 각 템플릿 표의 버튼 이름 그대로 (예: 대시보드 바로가기)
   - **Mobile 링크 (필수)** → 변수 쓸 때는 **`https://#{링크}`** 또는 **`https://#{자세히보기}`** 등 **프로토콜 포함**해서 입력 (안내에 프로토콜 필수)
   - **PC 링크 (선택)** → Mobile과 동일 (예: `https://#{링크}`) 또는 비워도 됨
7. 버튼 저장 후 **즉시 심사 요청**
