# 메타(Meta for Developers) 앱 기본 설정 — URL·네임스페이스 안내

앱 제출 시 **개인정보 처리방침 URL**, **서비스 약관 URL**, **네임스페이스** 입력 방법입니다.

**운영 사이트:** [https://www.onemarketing.kr/](https://www.onemarketing.kr/) (여기 기준으로 URL 입력)

---

## 1. 개인정보 처리방침 URL

**필드:** 기본 설정 → **개인정보처리방침 URL**

아래 **운영 URL**을 그대로 입력합니다.

| 구분 | 입력할 URL |
|------|------------|
| **운영 (메타 제출용)** | **`https://www.onemarketing.kr/privacy`** |

---

## 2. 서비스 약관 URL

**필드:** 기본 설정 → **서비스 약관 URL**

아래 **운영 URL**을 그대로 입력합니다.

| 구분 | 입력할 URL |
|------|------------|
| **운영 (메타 제출용)** | **`https://www.onemarketing.kr/terms`** |

---

## 3. 사용자 데이터 삭제 (데이터 삭제 안내 URL)

**필드:** 기본 설정 → **사용자 데이터 삭제** → **데이터 삭제 안내 URL** (또는 "링크를 제공할 수도 있습니다" 입력란)

Meta 정책상, 앱이 사용자 데이터를 저장할 경우 **데이터 삭제 요청 방법**을 안내하는 URL을 제공해야 합니다. 사용자가 "앱 제거" 등을 할 때 이 페이지로 안내됩니다.

| 구분 | 입력할 URL |
|------|------------|
| **운영 (메타 제출용)** | **`https://www.onemarketing.kr/data-deletion`** |

- 이 프로젝트에는 **`/data-deletion`** 공개 페이지가 있습니다.
- "회원 탈퇴로 삭제", "이메일로 삭제 요청", "Facebook·Meta 연동 데이터 삭제" 방법을 안내합니다.

---

## 4. 네임스페이스(Namespace)란?

**필드:** 기본 설정 → **네임스페이스**

- **네임스페이스**는 Meta 앱의 **고유 식별자**입니다.
- 주로 **Open Graph**에서 사용합니다.
  - 예: 커스텀 스토리·액션을 쓸 때 `네임스페이스:액션` 형태(예: `onemarketing:watch`)로 앱을 구분합니다.
- **Facebook 로그인만 쓰고**, Open Graph 스토리·커스텀 객체/액션을 쓰지 **않는다면** 비워 둬도 되는 경우가 많습니다.
- 다만, 앱 유형이나 제출 단계에서 **필수**로 표시되면 입력해야 합니다.

### 입력할 값 (필수인 경우)

- **영문 소문자, 숫자, 언더스코어(_)** 만 사용 가능.
- 앱 표시 이름과 맞추려면 예: **`onemarketing`**
- 다른 앱과 겹치지 않는 **고유한 문자열**이어야 합니다. (Meta 전역에서 유일)
