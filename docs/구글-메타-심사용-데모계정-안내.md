# 구글·메타 앱 심사용 데모 계정 안내

Google / Meta 앱 심사 제출 시 **심사관이 로그인해 실제 서비스 화면을 확인**할 수 있도록 제공하는 테스트 계정 안내입니다.

---

## 1. 데모 계정 정보

| 항목     | 값 |
|----------|----|
| **이메일**   | `demo@gmail.com` |
| **비밀번호** | `Admin123!` |

- 위 계정으로 **로그인 URL**(서비스 로그인 페이지)에 접속 후 로그인하면 됩니다.
- 이메일 인증이 완료된 상태이므로 별도 인증 절차 없이 바로 사용 가능합니다.

> **참고**  
> 실서비스 배포 시에는 공개 문서에 비밀번호를 노출하지 말고, **심사 제출 시에만** 심사 플랫폼(Google Play Console, Meta for Developers 등)의 테스트 계정 입력란에 위 정보를 기입하거나 별도 문서로 전달하는 것을 권장합니다.

---

## 2. 로그인 후 확인 가능한 화면

데모 계정으로 로그인하면 **빈 화면이 아닌 샘플(목업) 데이터**가 채워진 상태로 보입니다.

| 메뉴 / 화면   | 확인 가능한 내용 |
|---------------|------------------|
| **개요**      | KPI 카드(방문자 수, 리드 수, 전환율, 광고비), 성과 그래프(일자별 추이), 최근 실행 내역, 최신 리포트, 캘린더 로드맵 |
| **실행 현황** | 실행 항목 목록 및 상세(설명, 관련 링크, 마감일, 관련 프로젝트) |
| **캘린더**   | 일정 목록(회의, 점검, 오픈 등) |
| **프로젝트** | 프로젝트 목록(랜딩, 웹사이트, 프로모션 등) 및 진행 단계 |
| **리포트**   | 주간/월간 성과 리포트 목록 및 상세 요약 본문 |

- 리포트·파일 다운로드 링크는 **데모용 경로**이며, 실제 파일이 업로드된 것은 아닐 수 있습니다.
- 성과 그래프·KPI는 **목업 데이터**로, 실제 광고/분석 연동 값이 아닐 수 있습니다.

---

## 3. 구글(Google) 심사 제출 시

- **Google Play Console** 또는 **Google API/ OAuth 동의 화면** 등에서 “테스트 계정”을 요구하는 경우, 위 **이메일**과 **비밀번호**를 해당 입력란에 기재하세요.
- “테스트 사용자로 로그인하여 [개요 / 실행 현황 / 리포트 등] 화면을 확인할 수 있다”는 식으로 **확인 가능한 기능**을 간단히 적어 두면 심사에 도움이 됩니다.

---

## 4. 메타(Meta) 심사 제출 시

- **Meta for Developers** 앱 심사에서 “앱 로그인 계정” 또는 “테스트 계정”을 요구하는 경우, 위 **이메일**과 **비밀번호**를 제출하세요.
- 로그인 URL과 함께 “해당 계정으로 로그인 시 개요, 실행 현황, 캘린더, 프로젝트, 리포트 등 실제 사용자 화면을 확인할 수 있다”고 안내하면 됩니다.

---

## 5. 데모 계정·데이터가 없을 때 (운영자용)

데모 계정이 아직 없거나, 계정은 있는데 화면에 데이터가 비어 있다면 아래 스크립트를 사용할 수 있습니다.

- **계정 + 목업 데이터 한 번에 생성**  
  `npm run seed:demo`  
  → 새 데모 사용자 생성 후, 같은 클라이언트에 목업 데이터 삽입.

- **계정은 그대로 두고 목업 데이터만 추가**  
  `npm run seed:demo:data`  
  → 기존 데모 클라이언트에 연결된 사용자에게 projects, actions, reports, KPI, 성과 그래프용 데이터 등만 추가.

자세한 절차와 환경 변수는 **[심사용 테스트 계정 (목업 데이터 포함)](./심사용-테스트-계정.md)** 문서를 참고하세요.
